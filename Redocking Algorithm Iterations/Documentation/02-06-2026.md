# Revision of the script for redocking.

The tools necessary were compiled upon installation of PopOS. Starting with the compiling of VinaGPU:
![VinaGPU](Images/VinaGPU%20Compilation.png)

It was compiled using CUDA release 12.0. Since compiling with 'anything but' caused major issues like segmentation faults. The compiling of VinaGPU also needed Boost 1.84.0, thus this dependency was downloaded as well. Eventually, the tools needed for the research were downloaded and placed in a specified folder `/home/li/pseudobin/appname.` So it can be easily implemented in pipelines. The applications/binaries used are indicated below:
    - NAnoscale Molecular Dynamics (NAMD) - [Linux-x86_64-multicore-CUDA](https://www.ks.uiuc.edu/Development/Download/download.cgi?PackageName=NAMD) - 
    - Virtual Molecular Dynamics (VMD) - [LINUX_64 (RHEL 8+) OpenGL, CUDA, OptiX RTX, RTX RTRT (Linux (RHEL 8+) 64-bit Intel/AMD x86_64 SSE/AVX+ with CUDA 10, OptiX6.5 RTX, RTX RTRT)](https://www.ks.uiuc.edu/Development/Download/download.cgi?PackageName=VMD)
    - [Gnina](https://github.com/gnina/gnina)
    - Vega ZZ - [Commandline Version](https://www.ddl.unimi.it/cms/index.php?Software_projects:VEGA_ZZ:Download)
    - ChimeraX - [ucsf-chimerax_1.11.1ubuntu24.04_amd64.deb](https://www.cgl.ucsf.edu/chimerax/download.html)
    - AutoDockTools from [MGLTools 1.5.7](https://ccsb.scripps.edu/mgltools/downloads/)

Binaries that were built in the commandline were miniconda (an environment manager), seqtk (used for managing DNA sequences), and datamash (for commandline management of data in Bash). Environment-dependent binaries such as AmberTools25, PDBFixer, OpenBabel, and Meeko. Incompatibility of Meeko's atomtypes for VinaGPU made it impossible to dock due to the version disparity between the two so the utilities from MGLTools' folder made it feasible for docking.

# Creation of the script

The draft script was written in between class hours with a provided ![video](). It was later revised at the reseachers' residence. The first script, Iteration_1.sh, was fairly accurate, however there were moments in the RMSD Raw data where the data would exhibit sporadic leaps in the data, an indicative of major deviation. A CNNscore criteria was later introduced to Iteration_2.sh, making it much more accurate than it 's previous predecessor, the poses are sometimes hallucinated by the AI, adding a filter was suppose to act as to "remove any lower confidence' comformations.
