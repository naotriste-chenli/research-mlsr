# Creation of the script for Redocking and its execution (02/06/2026, 02/07/2026)

The draft script was written in between class hours with a provided ![video](). It was later revised at the reseachers' residence. The first script, Iteration_1.sh, was fairly accurate, however there were moments in the RMSD Raw data where the data would exhibit sporadic leaps in the data, an indicative of major deviation. A CNNscore criteria was later introduced to Iteration_2.sh, making it much more accurate than it 's previous predecessor, the poses are sometimes "too unphysical" and deviant to the original pose marked by the "inf" RMSD evaluated by 'obrms,' and is actually caused by --cnn_scoring refinement and --cnn_scoring all, although both are powerful for refining the poses by using a Convoluted Neural Network (CNN), it is deemed too unstable. Adding of the filter was suppose to act as to "remove any lower confidence' conformations in the lower portion of the Iteration_2.sh script (excludes any values less than 0.8 CNNscore). Later on, Iteration_3.sh is more of an attempt to know the results if only --cnn_scoring rescore was used with the filter still active, but this time, it can tolerate CNNscores of > 0.7, the results were promising but lackluster to the trend that Iteration_2.sh was exemplifying. Finally, the best of both worlds was acquired in Iteration_4.sh, the researchers applied a self-depedent filtering pipeline where the RMSD is first computed to the minimized structure against the dock structures, if the difference between is "inf," the suspecting ligands are removed. It showed favorable results, but each with its own challenges, that being, the docking rate or success rates.
